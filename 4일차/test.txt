*git
=>파일 관리(내가 원하는 시점에 파일을 불러올 수 있음)
=>파일을 추적(git이 보면서)하여 파일을 기록함
=>개발자뿐만 아니라 다양한 분야에서 사용함 작가등
=>초급자/고급자용이 있다.


*git hub
=>저장소의 개념(클라우드 역할)
=>호스팅사이트
=>git을 사용한 기록을 UI로 표현해줌
=>배포 가능


*집에서 설치할 것
1. git을 구글에 검색해서 설치할 것
2. git hub desktop 검색해서 설치
3. 설치한 깃허브 hub desktop 열어서 로그인 및 인증하기
4. 저장소(레포지토리) 만들기
5. creat repositories ... 세번째 클릭
6. 창이 뜨면 name은 레포지토리 명 -> 하단에 저장할 경로 -> 레로지토리 생성
7. 생성하면 지정한 경로 폴로 안에 폴더를 vscode로 열기 (***지정한 폴더 안에 폴더로 열기!***)
8. 파일 만들고, 코드 작성하던데로 작성하기
9. 깃허브 데스크탑 열어서 좌측하단에 Summary (required) 부분에
   기록할 타이틀 작성 (커밋 과정)
10. 하단 commit to main 버튼 클릭하여 커밋(기록) 남기기
11. 기록을 업로드 할 때 상단 탭 메뉴 중 Publish repository 버튼 클릭하여
    업로드 시키기
12. 깃허브 페이지에서 새로고침하여 잘 업로드 되었는지 확인해보기!


*파일이 없거나 처음 설치 환경일 때
=>깃허브 사이트에서 내려 받고자 하는 레포지토리 클릭
=>중앙상단 <> Code 버튼을 클릭하여
=>Open with Github Desktop 클릭하여 프로그램 열기
=>경로 실종됐으면 clone으로 깃허브에 있는 파일 복제 하기
